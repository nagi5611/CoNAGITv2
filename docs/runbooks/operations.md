# Runbook: 日常運用・通知のひな型（抜粋）

実装計画書・要件の「運用に残す」項目のうち、**リポジトリに固定できる最小限**を置く。具体閾値・ダッシュボード URL は組織の監視基盤確定後に追記する。

## 課金・利用状況のモニタリング（プレースホルダ）

以下は **確認観点の列挙** であり、ツール名・閾値は環境ごとに設定する。

- [ ] **Cloudflare Workers**: リクエスト数、CPU 時間、エラー率（ダッシュボード／ログ）
- [ ] **D1**: 読み書きユニット、ストレージ増加率
- [ ] **S3**: ストレージ容量、GET/PUT 回数、データ転送（コスト異常の早期検知）
- [ ] **（将来）CloudFront**: 転送量・エラー（導入時に Runbook 更新）

詳細の思考整理は要件定義 §13 と [AWS S3 / CloudFront Runbook](./aws-s3-cloudfront.md) を参照。

## メンテナンス予告テンプレート（日本語）

件名: 【メンテナンス予告】CoNAGITv2（日時を記載）

本文例:

```
弊社（または◯◯部）で提供している CoNAGITv2 について、以下の日程でメンテナンスを実施します。

■ 日時: YYYY年MM月DD日(曜) HH:MM 〜 HH:MM（予定）
■ 影響: サービス全体または（対象範囲を記載）
■ 内容: （例：デプロイ、データベースメンテナンス、インフラ変更 等）

作業完了後、改めてご連絡する場合があります。お急ぎの場合は（連絡先）までご連絡ください。
```

※ 実際の連絡チャネル・承認フローは社内規程に従う。
