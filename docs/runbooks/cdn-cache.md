# CloudFront / CDN cache (Phase I stub)

> **状態**: フェーズ I（実装計画書 §4）の**プレースホルダ**。本番 CloudFront・署名 URL・`filename*` 等は AWS アカウントとゾーン設定が必要なため、本リポジトリではコード・Runbook の骨子のみ置く。

## 目的（計画書より）

- 署名付き取得、`Content-Disposition` / RFC 5987 `filename*` による日本語ダウンロード名。
- キャッシュ整合（イミュータブルキー優先、無効化の乱用禁止）。

## 運用ルール（ドラフト）

1. オブジェクトキーは可能な限り**内容ハッシュまたはバージョン付き**にし、更新時に URL が変わるようにする（ブラウザ・CDN の古い応答を避ける）。
2. グローバル無効化（Invalidation）は**リリースゲート承認後のみ**、対象パスを最小化する。
3. 設定変更（TTL、OAC、オリジン）は**ステージングで検証**してから本番。本番は二人確認（実装計画書 §5）。

## 実装メモ（未着手）

- Worker から CloudFront 署名 Cookie / URL を発行する場合の鍵ローテ手順は `docs/runbooks/aws-s3-cloudfront.md` と合わせて記載する。
- プレビュー URL とメタデータ更新の**整合性テスト**はステージングで実施（CI ではモックのみ）。
