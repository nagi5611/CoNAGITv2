# Runbook: インシデント対応（初動〜エスカレーション枠）

組織固有の連絡先・オンコール表は本リポジトリには含めない。**電話番号・チャット URL は社内 Wiki 等に記載**し、本書は手順の型のみ示す。

## 1. 重大度の目安（例）

| レベル | 例 |
|--------|-----|
| 高 | 認証不能、全ユーザーがファイル一覧を開けない、データ欠損の疑い |
| 中 | 一部グループのみ、読み取りは可だが書き込み不可 |
| 低 | UI のみ、ワークアラウンドあり |

## 2. 初動チェックリスト

1. **ユーザー影響**を簡潔に記録（開始時刻、再現手順があれば）。
2. **変更の有無**: 直近デプロイ・設定変更・IAM・DNS を確認。
3. **依存サービス**: Cloudflare（Workers/D1/Queues）と AWS（S3/CloudFront）のステータスページを確認（外部依存）。
4. **データ保全**: リストアやパージを伴う操作は Runbook を開いた状態で実施する。

## 3. エスカレーション（枠）

1. 一次: 当番オペレーターが初動・連絡・ログ収集。
2. 二次: テックリードまたはインフラ担当へ（設定・コード変更の判断）。
3. 経営・利用部門への連絡は社内規程に従う。
