# Runbook: AWS S3 と CloudFront（ドラフト）

詳細な初回セットアップ手順（バケット非公開、IAM、CORS、CloudFront とプリサインの関係）は **[マニュアル: AWS S3 + CloudFront セットアップ（入門）](../manuals/aws-s3-cloudfront-setup.md)** を参照してください。

実装計画書フェーズ B の成果物テンプレート。**本番／ステージングのバケット名・ディストリビューション ID は AWS 側で確定後に追記する**（実装計画書 R37: 環境取り違え防止のため、コンソール上のラベル色分け・命名規約を併用する）。

## 構成（概要）

- **S3**: 単一バケット、オブジェクトは非公開（要件定義 6.1）。
- **CloudFront**: オリジンは S3 のみ（要件定義 6.2）。OAC 等でオリジン直アクセスを遮断する。
- **誤ってパブリックにした場合**: AWS 公式の「パブリックアクセスのブロック」と Access Analyzer の手順に従い、即座にパブリック ACL／ポリシーを取り除く。詳細 URL は組織のセキュリティ Runbook に委譲。

## 完了条件（フェーズ B）

- パブリック直アクセス不可。
- 署名経路（署名付き URL または署名付き Cookie）での取得が可能。

## 実装との対応（コードベース）

| 観点 | 実装の事実 |
|------|------------|
| オブジェクト保管 | 単一バケット・プロジェクト単位プレフィックス（`buildS3ObjectKey`） |
| アップロード | Worker がプリサイン PUT／マルチパート URL を発行し、クライアントが S3 に直接 PUT（`src/routes/upload.ts`） |
| 取得・プレビュー | Worker が **S3 向けプリサイン GET** を発行（ダウンロード・画像プレビュー）。CloudFront オリジンへ差し替える場合は別 ADR／Runbook 更新が必要（[ADR 0005](../decisions/0005-cdn-s3-delivery.md)） |
| キャッシュ無効化 | `CDN_INVALIDATION_WEBHOOK_URL` 等が設定されている場合のみ通知（未設定は no-op） |

※ **架空の AWS アカウント ID・バケット名・ディストリ ID は本文に書かない**（運用で確定した値のみ追記）。
