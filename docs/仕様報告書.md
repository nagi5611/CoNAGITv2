## 1. はじめに

### 1.1 本書の目的

本書は、当プロジェクトで構築する**社内向けストレージ共有サービス**が「何を目的とし、どのような形で提供されるか」を、関係者が短時間で把握できるようにまとめたものである。画面遷移や API の列挙などの詳細は扱わず、**全体像と主要な振る舞い**に限定する。

### 1.2 想定読者

プロジェクト責任者、関係部門のレビュー担当、実装チームのキックオフ参加者。

---

## 2. システム概要

本システムは、**1 社の組織の中**で、**グループ**と**プロジェクト**に分かれてファイルを管理する Web アプリケーションである。利用者はブラウザから、エクスプローラに近い操作感でフォルダ・ファイルの作成・アップロード・移動・削除を行い、画像・動画などの**プレビュー**や、一定の**テキストファイルの編集**ができる。

ユーザーの追加・削除は**管理者**が行う限定的な運用とし、各グループでは**グループリーダー**が所属メンバーを調整する。ファイルの実体は **Amazon S3** に置き、名前や階層・権限などの情報は **Cloudflare D1** に保持する**二層構成**とする。配信は **Amazon CloudFront** を用い、署名付き URL や Cookie でアクセスを制御する想定である。

**実際の Git リポジトリ機能（clone や branch など）は持たない。** あくまで「Git やエクスプローラに近い見た目・操作性」を参考にしたファイル管理 UI である。

---

## 3. 想定ユーザーと役割

| 役割 | 概要 |
|------|------|
| 管理者 | 会社ユーザーの登録・削除、グループリーダー指名、ゴミ箱の空化・完全削除、通常領域からの**完全削除**など。 |
| グループリーダー | 当該グループへの一般ユーザーの割当（登録済みユーザーから選択・検索）。プロジェクトの削除（ルールに従う）。ファイル／フォルダ削除は**ソフト削除**のみ。 |
| 一般ユーザー | 所属グループ内でプロジェクト作成、フォルダ・ファイル操作、他人ファイルの削除（ソフト削除）など。信頼された社内利用を前提とする。 |
| 未割り当て | ログインはできるが、グループ未所属のためプロジェクト一覧は表示しない。 |

---

## 4. 主な機能（概要）

1. **認証** … フォームによる ID・パスワード入力とセッション Cookie。
2. **プロジェクト一覧** … 自分が所属するグループ配下のプロジェクトのみ。複数グループ所属時は一覧で横断表示。
3. **ファイルツリー** … フォルダの作成・削除・移動、ファイルのアップロード・削除・ダウンロード。S3 直アップロード（大容量はマルチパート等、要件定義に従う）。
4. **プレビュー** … 画像・動画等。3D はアイコン中心の UI。
5. **テキスト編集** … ブラウザ上エディタ。UTF-8 と Shift_JIS。保存時に監査ログ（差分は記録しない）。
6. **プロパティ** … 右クリックでメタ情報・監査・リネーム履歴などを参照。
7. **ゴミ箱（グループ単位）** … 一般ユーザー・グループリーダーの削除は原則 1 週間保持。全員が閲覧可能。空化・中身の完全削除は管理者のみ。
8. **サムネイル** … 画像の縮小版・動画の先頭フレーム相当を生成し S3 に保存。**非同期キュー**で生成し、失敗時はプレースホルダ表示にフォールバック。

## 5. 技術構成（概要）

| 区分 | 採用の考え方 |
|------|----------------|
| アプリ・API | Cloudflare Workers（認可、メタデータ、署名 URL 発行など） |
| メタデータ DB | Cloudflare D1 |
| 非同期処理 | Cloudflare Queues（サムネイル生成など） |
| オブジェクトストレージ | AWS S3（バケット 1 つ、プロジェクト単位プレフィックス） |
| CDN・ダウンロード制御 | Amazon CloudFront（オリジン S3、署名付き URL / Cookie） |
| 画像・動画変換 | Cloudflare の Image / Media Transformations を利用する方針（要件定義の出典リンク参照） |
