# ADR 0002: オブジェクト保管を S3 としクライアントは署名 URL で PUT する

| 項目 | 内容 |
|------|------|
| ステータス | 提案 |
| 日付 | 2026-05-10 |

## コンテキスト

ファイル本体は非公開バケットに置き、ブラウザやクライアントから直接 Workers に巨大ボディを流し込むとタイムアウト・メモリ・コストのリスクがある。代替として Workers がプロキシ転送する方式や、Multipart を Workers 完結で処理する方式も考えられる。

## 決定

オブジェクトキーとメタデータは D1 に登録し、**アップロード経路は AWS SigV4 のプリサインド PUT（およびマルチパート制御 API）** とする。クライアントは署名された URL へ直接 PUT し、完了後に Workers の commit API でサイズ等を確定する。

## 結果

### ポジティブ

- Workers が大容量バイトストリームを保持しない。
- S3 のマルチパート・整合性モデルをそのまま利用できる。

### ネガティブ / リスク

- IAM・バケットポリシー・CloudFront/OAC の設定ミスはデータ露出に直結する（別 Runbook で監査）。
- 署名 URL の有効期限・キー設計をコードと運用で一致させる必要がある。

### フォローアップ

- `docs/runbooks/aws-s3-cloudfront.md` と署名実装（`src/s3/`）をレビュー時にセットで確認する。
