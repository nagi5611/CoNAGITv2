# ADR 0001: ランタイムを Cloudflare Workers + D1 とする

| 項目 | 内容 |
|------|------|
| ステータス | 提案 |
| 日付 | 2026-05-10 |

## コンテキスト

社内向けファイル共有では、メタデータ（ユーザー・グループ・ツリー・ゴミ箱・監査）を一貫して扱いつつ、オブジェクト本体は AWS S3 に置く要件がある。API のホスティング先として、従来型 VPS・コンテナ・サーバーレス（Lambda 等）も選択肢となる。

## 決定

HTTP API とバッチ的な内部処理の中心を **Cloudflare Workers** とし、リレーショナルメタデータは **D1（SQLite）** に置く。Workers と同一ベンダー内で低遅延にクエリでき、Wrangler によるデプロイ・シークレット・キュー連携と整合させやすい。

## 結果

### ポジティブ

- エッジ近傍での応答と、D1 との単一スタック運用。
- `wrangler` による環境間の差分管理がしやすい。

### ネガティブ / リスク

- SQLite/D1 の運用特性（バックアップ・リストア、同時書き）をチームで理解しておく必要がある。
- AWS 側（S3・CloudFront）は別アカウント設定が前提となる。

### フォローアップ

- バックアップ・リストア手順は `docs/runbooks/d1-backup-restore.md` と整合させる。
