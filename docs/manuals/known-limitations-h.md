# フェーズ H（Web MVP）の既知の制限

根拠: [仕様報告書](../仕様報告書.md)、[実装計画書](../実装計画書.md) フェーズ H、リスク登録簿 R23〜R24・R55・R57〜R58。

## ブラウザ・端末

| 項目 | 内容 |
|------|------|
| 動作確認の主対象 | Chrome / Edge / Safari の最新版を想定（実装計画書 §5）。Firefox は任意。 |
| モバイル | デスクトップ優先の MVP。狭い画面ではレイアウトが窮屈になる可能性がある（R58）。 |
| Safari / ITP | 同一サイト Cookie 設計に依存。ログインが不安定な場合はドメイン・Cookie 属性の Runbook を参照（R57、[セッションと Cookie](../security/sessions.md)）。 |

## 機能・UX

| 項目 | 内容 |
|------|------|
| ツリー規模 | 巨大なフォルダ階層・ファイル数では描画・スクロールが重くなる可能性（R24）。必要なら運用でフォルダ分割を推奨。 |
| エラー表示 | API が返す `error.message` をそのまま表示する箇所がある。英語メッセージが混在する場合はバックエンド文言整備が必要（R55）。 |
| プレビュー | 画像はブラウザがプリサイン URL に直接 GET する。S3 バケット CORS が不適切だと表示されない（[操作ガイド](./user-guide.md)）。URL の寿命は `PRESIGN_GET_EXPIRES_SECONDS`（既定 120 秒）に依存し、期限切れ後は再取得が必要。 |
| 動画・特殊コーデック | ブラウザ内プレビューは画像・許可されたテキスト系に限定。動画インライン再生などは対象外。 |

## セキュリティ・権限（UI 観点）

| 項目 | 内容 |
|------|------|
| 完全削除 | Web からは会社管理者にボタンが出る設計。誤操作防止のため確認ダイアログあり（[操作ガイド](./user-guide.md)）。 |
| 監査ログ | 会社管理者のみ UI から閲覧。一般ユーザーは API 直接叩きでも権限で遮断される（[権限マトリクス](./permissions-matrix.md)）。 |
