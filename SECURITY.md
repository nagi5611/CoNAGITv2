# セキュリティ

## 脆弱性の報告

セキュリティ上の問題を発見した場合は、**公開 Issue に詳細を書かず**、社内の定めた連絡経路（セキュリティ担当・テックリード）へ報告してください。連絡先が未整備の間はリポジトリ管理者へ直接連絡してください。

## 依存関係のレビュー（リリース前の目安）

実装計画書フェーズ M・リスク R52 / R73 に沿い、次を推奨します。

| 手段 | 備考 |
|------|------|
| `npm audit` | 既知の npm 依存の脆弱性の一次スクリーニング。 |
| lockfile | `package-lock.json` をコミットし、CI で `npm ci` と整合させる。 |
| 更新方針 | Dependabot 等の自動 PR と、破壊的更新の手動レビューのバランスをチームで決める。 |

本リポジトリは **本番 AWS / Cloudflare アカウントへの接続を前提としない** CI（dry-run・Vitest）を主とする。実環境へのペネトレーションや本番データを用いた検証は別手順・承認のうえで実施してください（実装計画書 R80）。
